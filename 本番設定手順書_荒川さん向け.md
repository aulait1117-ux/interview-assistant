# 面接アシスタント 本番設定手順書

対象者：荒川さん  
作業時間：約30分  
作成：CTO AI（2026-06-30）

---

## 作業前チェック

- [ ] Stripeアカウントの本人確認（KYC）が完了している
- [ ] Stripeに銀行口座が登録されている
- [ ] Renderのダッシュボードにアクセスできる

---

## STEP 1：Stripe 本番キーを取得する

1. [https://dashboard.stripe.com](https://dashboard.stripe.com) にログイン

2. 右上のトグルを **「本番環境」** に切り替える（初期はテストモード）

3. 左メニュー「開発者」→「APIキー」を開く

4. 以下をメモしておく：
   ```
   シークレットキー：sk_live_○○○○○○○○
   ```
   ※ 公開可能キー（pk_live_...）は今回不要

---

## STEP 2：Stripe Webhook を登録する

1. 左メニュー「開発者」→「Webhook」を開く

2. 「エンドポイントを追加」をクリック

3. 以下を入力：
   ```
   エンドポイントURL：
   https://interview-assistant-hrar.onrender.com/api/billing/stripe/webhook
   ```

4. 「リッスンするイベント」で以下の3つを選択：
   - `checkout.session.completed`
   - `checkout.session.async_payment_succeeded`
   - `checkout.session.async_payment_failed`

5. 「エンドポイントを追加」をクリック

6. 作成されたwebhookの「署名シークレット」を表示して以下をメモ：
   ```
   Webhookシークレット：whsec_○○○○○○○○
   ```

---

## STEP 3：Render に環境変数を設定する

1. [https://dashboard.render.com](https://dashboard.render.com) にログイン

2. サービス一覧から **`interview-assistant-backend`** を選択

3. 左メニュー「Environment」を開く

4. 以下の環境変数を追加（「Add Environment Variable」ボタンを繰り返す）：

   | キー名 | 値 |
   |--------|-----|
   | `STRIPE_SECRET_KEY` | STEP 1 でメモした `sk_live_...` |
   | `STRIPE_WEBHOOK_SECRET` | STEP 2 でメモした `whsec_...` |
   | `ANTHROPIC_API_KEY` | （社長または開発部に確認） |
   | `OPENAI_API_KEY` | `sk-svcacct-kv3gVHRRG_gp1Os...`（`.env`ファイルと同じ値） |
   | `GOOGLE_CLIENT_ID` | （Google Cloudコンソールで確認） |

5. 「Save Changes」をクリック → 自動でデプロイが始まる（2〜3分待つ）

---

## STEP 4：デプロイ完了を確認する

1. Renderダッシュボードの「Deploys」タブを開く

2. 最新のデプロイが **「Live」** になるまで待つ

3. バックエンドの疎通確認：  
   ブラウザで以下のURLを開いて `{"status":"ok"}` が返ってくることを確認：
   ```
   https://interview-assistant-hrar.onrender.com/health
   ```

---

## STEP 5：テスト決済を行う（実際のカードで）

> ⚠️ ここから**本物のお金**が動きます。小額（¥500）でテストして返金します。

1. 面接アシスタントアプリを起動してログイン

2. 料金プランページで **「1日1時間プラン（¥500）」** を選択

3. 「カード / Apple Pay / Google Pay」を選んで購入ボタンをクリック

4. 実際のクレジットカードで決済

5. 以下を確認：
   - [ ] Stripeダッシュボードに¥500の入金記録がある
   - [ ] アプリでプランが有効になっている
   - [ ] Webhookログに `checkout.session.completed` が記録されている

6. Stripeダッシュボードから返金処理を行う：
   - 「支払い」→ 該当トランザクション → 「返金」

## トラブルシューティング

| 症状 | 確認箇所 |
|------|---------|
| Webhook失敗（400エラー） | STRIPE_WEBHOOK_SECRET の値が正しいか確認 |
| 決済後にプランが有効にならない | Renderのログ（Events → Logs）を確認 |
| バックエンドに接続できない | Renderの「Deploys」タブでデプロイが成功しているか確認 |
| `checkout.session.completed` が届かない | Webhookの登録URLが正確か確認 |
